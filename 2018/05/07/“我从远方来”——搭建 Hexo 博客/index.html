“我从远方来”——搭建 Hexo 博客

发表于 2018-05-07 | 更新于 2018-07-29 | 分类于 经验

之前由于 Hexo 博客没有搭建好，一部分文章就被放在了其他地方，现在搬过来。 这是一篇从5月4日写到了5月5日的文章，它记录了我搭建现在这个博客的全过程，以及其他。

首先，要向师兄的耐心表示敬意

上周日正式开始用 Hexo + Github Pages 搭建博客，中途遇到了不少问题。林师兄一直不厌其烦地给我纠正错误。每次找到症结之后都会觉得自己有点傻，一点点小错误引发的后果很麻烦，幸好师兄没有嘲笑我，否则我铁定倍受打击再不想碰这个东西了。

中途遇见的问题的详细描述放在后面一起说吧~

简化的步骤

搭建 Node.js 环境

  • 下载+安装 Node.js
  • 在桌面空白处按 Shift + 右键打开命令窗口，输入node -v，按下回车键（即运行指令“node -v”），然后运行npm -v，检查版本和安装情况

搭建 Git 环境

  • 下载+安装 Git，注意勾选 on the desktop 和 use a true type font
  • 桌面右键 打开 Git Bash Here，运行git --version，以检查安装情况

GitHub 注册和配置

因为之前已经注册过 GitHub，所以我只需要添加仓库就好了。没有注册则需先注册再添加仓库。并且注意用户名只支持英文。

  1. 在 GitHub 官网里面点击头像，选择 Your profile
  2. 点击 Repository 新建仓库yourname/yourname.github.io，（yourname 指代你自己的用户名，千万不要直接填 yourname 啦！！下同）在 Repository name 中填“yourname.github.io”，Description 是选填项

Hexo 的安装和配置

  • 在 Git Bash Here 中先后运行命令npm install hexo-cli -g和hexo version检查版本和安装情况
  • 初始化 Hexo（如无特殊说明，都是在桌面打开 Git Bash Here）：
    1. 运行hexo init yourname.github.io（修改用户名）
    2. 运行cd yourname.github.io（修改用户名）
    3. 运行npm install，完成后会有新建的指定文件夹目录出现，它长这样：（#表示解释文件目录的内容）
├── .deploy         #需要部署的文件
├── node_modules    #Hexo插件
├── public          #生成的静态网页文件
├── scaffolds       #模板
├── source          #博客正文和其他源文件
| ├── _drafts       #草稿
| └── _posts        #博客文章
├── themes          #主题
├── _config.yml     #全局配置文件
└── package.json    #npm依赖
  1. 接下来运行hexo s，如无意外，应该就能在本地运行 Hexo 了（当时在自己的电脑上看见那个日出还是什么的时候好激动啊啊）
  2. 检查前面步骤，看看是否能打开本地运行 https://localhost:4000/ 如果能看见日出的话——意味着前方一片光明对我来说就是黑暗的开始

关联 Hexo 和 GitHub Pages

  • 在任意地方新建一个文件夹（要注意的是，这个文件将会是你以后存放个人博客所有信息的地方，所以位置和命名需要一丢丢思考）
  • 在刚才新建的文件夹的空白处打开 Git Bash Here 命令窗口，输入ssh -T git@github.com，得到反馈 (yes/no) 后输入 yes
  • 配置个人信息：输入git config --global user.name "yourname"（改用户名）
    和git config --global user.email "@qq.com"(补全 QQ 邮箱）

  • 配置 Deployment：查找文件_config.yml（比如我在D盘新建文件夹，将其命名为blog，那么这个文件的位置就是 D:\blog\lerinaz.github.io\_config.yml）用记事本打开
    找到这一段并修改为如下：

    # Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:yourname/yourname.github.io.git
  branch: master

要注意冒号后有空格；并且注意修改用户名

  • 最后一步，将本地文件提交到 GitHub Pages:
    在自动生成的yourname.github.io文件夹下（比如我的是 blog\lerinaz.github.io）运行指令hexo g，然后hexo d（据师兄说，也可以hexo g && hexo d 一起输入，实践证明这更方便，但是要小心这里是两个&）
  • 由于在我的安装过程中以上两个指令失败了，所以还需要安装一个扩展npm install hexo-deployer-git --save然后重复hexo g && hexo d
  • 这个时候在浏览器中输入https://yourname.github.io （修改用户名~）如果能打开的话，就说明一切都完成啦！

我遇到的问题

  • 第一次试水是4月29日，时间有点久远……当时是没有新建文件夹导致命令都没法运行。
  • 第二次，也就是昨天，前面都很顺利，而在最后一步 Deploy 出了状况，因为理解错了意思，也忘了#表示说明，把三行命令一起输入了，现在想起来真是尴尬。

后话

到这里就全部完成了，但是接下来一个更加艰难的问题摆在了我的面前——搭好之后呢？师兄emm了一下，说：你可以写篇文章庆祝一下，可能他还不知道我根本不知道要怎么用吧……

回过头来

在外面跑了一个下午之后，昨晚接着做的。可能是之前已经犯过错的缘故，重复第上次的步骤时都比较顺利。直到最后一步，完全走不了了。当我问师兄哪里不对时，他告诉我“没错啊，你不是一次过了吗？”大佬真看得起我啊，这中间的每一次障碍如果没有他，我一个都解决不了。我又说本地文件提交到 Github 那里怎么不对，他依然是那句话“你只是忘了部署，总体来说是一次过。”这，这真是养孩子一样的鼓励啊……后来还有好多次理解错他的意思，比如让我hexo deploy，问我之前安装成功了吗？我以为问的是 Hexo——因为解答疑惑的时候告诉我说包管理器自动处理，还反问我“不然你之前怎么安装的 Hexo？”联系前后文，我就以为这里问的是 Hexo 有没有安装成功。感觉我的大脑已经习惯于散漫的诗歌散文和明清小品，而常常理解错指令的意思了。

中间每一步都是截图，还是复制指令，因为我照着打的命令都常常出错。最重要的一句话一旦有报错就不要继续往下，先解决错误。（因为之前自己做的时候遇见不对的地方都没管它，只要没有红字我都接着走。中途师兄说：继续，每一步截图，哇感觉自己受到帝王级的待遇，竟然让信软大佬一步一步地教。而之前自己照着官网说明做的时候，都没怎么看反馈，一通乱走，到最后啥也不知道，也许从很早以前就出错了。

主要技术支持来自师兄，还参考了两位博主的博客：白小明和Xuanwo

向三位拜谢~~

写这篇文章时，看见美丽的蓝花楹

上午都在图书馆，湖边的蓝花楹全开了，这才知道那棵树竟是蓝花楹。

想起去年冬天，在为期末考试复习的那段时间里，从早到晚都窝在图书馆，每隔一个小时就到落地窗这里来望望远处，当时看见它光秃秃的，还以为是棵栾树来着。

这是我第一次看见蓝花楹开花的样子，和想象中是一样的温柔浪漫，只是没想到它的花只能从上面看，因为柔软的蓝色花瓣都开在树冠，在树荫下是看不见的。从去年进校看见它们奇幻的树冠和叶片开始，我就盼望着花开的时节，而今年温暖的四月里，终于等到了。远远地望见组团旁的那棵蓝花楹的时候，会觉得整颗心都被它们融化了，像悠闲地躺在白云上，又会不由自主地想象着，在静谧的夜里，溶溶月色下倦鸟居巢，人闲无声，一片花瓣飘落的声音。图书馆旁边的那一棵和远处的主楼、树林形成了绝佳的组合景色，树林那浓密的绿色层层叠叠，掩映其中的是主楼洁白端庄的身姿，右边还有小小的西湖一角，有时候在阳光下泛着金色光芒。此时，蓝花楹温柔的紫色就成了点睛之笔，将入眼的画面全都点染上轻盈的梦，好像身在幻境之中。
