修改文章标签的大小写之后……

发表于 2018-05-09 | 更新于 2018-07-29 | 分类于 经验

由于排版不当，有些文章展示出来不怎么好看，所以昨晚挨个儿清查了一遍，并且将文章的 tag 由小写改为大写。然而打开博客的 Hexo 标签后却出现了404页面。这篇文章记录了解决这个问题的全过程。

当我知道中英文混排需要加空格时，我是没有想到此次修改所引发的一系列问题的。当时只想着“哇改了之后变好看了”，结果等我点开 Hexo 标签：

发生了什么？

最先想到的是可能是因为缓存，等一会儿就会好了。可是等了三会儿也是这样的结果，我有点担心，好像哪里不对劲。

然后点文章底部的标签也是这样，看来是真的出问题了。在海螺林的帮助下找到了原因：Windows 系统下的 Git 默认设置对大小写不敏感，在 GitHub 上展示出来的仍然是小写：

解决方案

于是按照师兄提供的解决办法进行修改：Hexo部署到GitHub大写开头的标签出现404（以下步骤引自该文章）

  1. 在博客文件夹中找到.deploy_git文件夹，打开config文件，将其中的ignorecase=true改为ignorecase=false

  2. 在.deploy_git文件夹中，删除里面的所有文件（注意不是删除.deploy_git文件夹，而是删除它里面的内容），并 push 到 Github 上, 这一步是清空你的 github.io 项目中所有文件。运行命令：

    cd .deploy_git

git rm -rf *

git commit -m 'clean all file'

git push

  3. 最后使用 Hexo 再次生成及部署。运行命令：

    cd ..

hexo clean

hexo g 

hexo d

我的状况

第一步就出事了：当我打开config文件时，发现ignorecase后面并不是接的true，而是ignorecase = true[branch master]，嗯——怎么办呢？先不管它，只把true改掉，方括号保留。然后运行命令，得到的反馈是：

看来不对，于是又把[branch master]去掉，仍然不对：

细读之下我注意到了bad numeric config value和invalid unit，这也许是说config文件中的格式出现了错误？并且由反馈来看，两个项目ignorecase = false和remote = git@github.com:lerinaz/lerinaz.github.io.git粘连到了一起，变成了remote = git@github.com:lerinaz/lerinaz.github.io.git，所以我推断ignorecase项和remote项之间的空格不对——或许是因为删除时的空格没留够？但是第一次只改没删也是这样的错误啊？

于是开始四处寻找，输入bad numeric config value和invalid unit，知道了是 Git 运行时由于其下的文件出错而无法识别，然后仔细对比，先是发现等号两边没有空格，然后又发现因为remote = git@github.com:lerinaz/lerinaz.github.io.git恰好在等号处换行，所以我又一次在换行开头时漏掉了空格。改掉这两处错误后还是不对，仍然是bad numeric config value，到此我已经花了一个上午搞这个东西，被它折腾到绝望。

睡个午觉起来接着弄，一次只加一个空格，加一个空格就删除并重新部署一次，在加到足足八个空格的时候我火了——搞什么呢？！为什么总是出问题！就不能一劳永逸吗？气得我反手打开了 B 站，看起布袋戏来。却又总想着这件事还没有解决掉，一阵纠结后又回到了config。不过这次我豁出去了，反正有这么厉害的林师兄在，干脆一不做二不休，直接把remote提行，要是翻车了就呼救……抱着这样的心态我大胆试了试，结果成功了！再打开博客标签：

但是，事情到这里并没有结束……

在添加评论区的时候又翻车了。修改评论区所对应的config文件之后出现了网页加载错误的问题，最终把症结定位在我之前对ignorcase所做的改动。因为当时是用记事本打开了文件，所以每一项应该有的提行并没有展示出来。在这里师兄告诉我“不要用记事本编辑任何配置文件”，并且让我以后都换做 VSCode。当我用 VSCode 打开config并再次修改的时候，果然清爽了很多：

并且能够很清楚地看见第 8 行的remote格式不对。也就是说，由于我之前用记事本做修改全凭感觉，导致现在出问题了。

所以

  • 更改config文件时注意符号两边是否需要空格
  • ignorecase = false后面到底要不要跟[branch master]呢？现在我也不知道，之后或许会试试吧
  • 上面一项被否决了，我也不敢再轻易尝试
  • 别用记事本打开config文件
